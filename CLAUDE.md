# pgfusion-benchmark

Standalone benchmark suite comparing pgfusion against PostgreSQL on industry-standard workloads.

## Structure

```
pgfusion-benchmark/
├── justfile            # task runner (requires just)
├── bench_lib.sh        # shared runner library (sourced by run.sh scripts)
├── clickbench/         # ClickBench — 43 analytical queries on the hits dataset
│   ├── setup.sh        # download dataset and load into PostgreSQL
│   ├── run.sh          # run benchmark and produce results
│   ├── queries.sql
│   └── checkpoints/    # saved run results (current/ + named slugs)
└── tpch/               # TPC-H — 22 queries at configurable scale
    ├── setup.sh        # generate data with dbgen and load into PostgreSQL
    ├── run.sh          # run benchmark and produce results
    ├── queries.sql
    └── checkpoints/    # saved run results
```

## Prerequisites

- `PG_HARNESS_DIR` set to your `pg-test-harness` clone (config derived automatically)
- `PGFUSION_ROOT` set to the `pgfusion/` crate root (builds `pgfusion_cli` from source; falls back to PATH)
- PostgreSQL installed and configured (run `just pg-setup` in `pgfusion/`)

## Configuration

```bash
export PG_HARNESS_DIR=/path/to/utils/pg-test-harness
export PGFUSION_ROOT=/path/to/pgfusion
```

This is the same file generated by `just pg-setup <version>` in `pgfusion/` or `pg_arrow/`.

## Running Benchmarks

```bash
# ClickBench
just clickbench-setup            # first-time setup (~75GB dataset)
just clickbench                  # run all 43 queries
just clickbench pg18 3 query=13  # single query
just clickbench-checkpoint pg18 3 before-opt  # run + save checkpoint

# TPC-H
just tpch-setup                  # first-time setup (default SF10)
just tpch                        # run all 22 queries
just tpch-query 7                # single query
just tpch-checkpoint pg18 3 baseline
```

## Checkpoint System

Every run writes to `<bench>/checkpoints/current/` (CSV + JSON + heatmap).
Named checkpoints save to `checkpoints/<git-short-hash>[-label]/`.

```bash
just clickbench-save my-label    # archive current without re-running
just clickbench-report           # open latest heatmap in browser
just clickbench-report-checkpoint f85939b-before-opt
```

## bench_lib.sh Variables

Each `run.sh` sets these before sourcing `bench_lib.sh`:

| Variable | Description |
|---|---|
| `SCRIPT_DIR` | Absolute path to the benchmark dir (`clickbench/` or `tpch/`) |
| `CONFIG_FILE` | Path to `pg-test-config.toml` (from `PG_ARROW_TEST_CONFIG`) |
| `PG_VERSION` | Version key, e.g. `pg18` |
| `BENCH_DB` | Database name (`clickbench` or `tpch`) |
| `BENCH_TITLE` | Summary heading |
| `BENCH_QUERY_NAME_STRIP` | Strip `:.*` suffix from query names (tpch only) |
| `BENCH_RESULTS_IN_ROOT` | Results in `$SCRIPT_DIR` vs `checkpoints/current/` |
| `BENCH_TUNE_WARN` | Warn when `max_parallel_workers_per_gather < 10` |
