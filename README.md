# pgfusion-benchmark

Benchmark suite comparing pgfusion against PostgreSQL on industry-standard workloads.

## Structure

```
pgfusion-benchmark/
├── bench_lib.sh        # shared runner library (sourced by run.sh scripts)
├── clickbench/         # ClickBench (43 analytical queries, hits dataset)
│   ├── setup.sh        # download dataset and load into PostgreSQL
│   ├── run.sh          # run benchmark and produce results
│   ├── queries.sql
│   └── checkpoints/    # saved run results
└── tpch/               # TPC-H (22 queries, scale-configurable)
    ├── setup.sh        # generate data with dbgen and load into PostgreSQL
    ├── run.sh          # run benchmark and produce results
    ├── queries.sql
    └── checkpoints/    # saved run results
```

## Prerequisites

- PostgreSQL installed and configured in `pg-test-config.toml`
- `pgfusion_cli` in PATH **or** `pgfusion` crate buildable via `cargo`
- `psql` and `pg_ctl` reachable from the paths in `pg-test-config.toml`

## Configuration

### Config file

Scripts require `PG_ARROW_TEST_CONFIG` pointing to `pg-test-config.toml`:

```bash
export PG_ARROW_TEST_CONFIG=/path/to/pgfusion/pg-test-config.toml
```

This is the same file generated by `just pg-setup <version>` in either crate.

### Binary

If `pgfusion_cli` is in PATH it is used directly (no build step).
Otherwise scripts fall back to `cargo build --release` inside the `pgfusion` crate.

## Usage

### ClickBench

```bash
# First-time setup (downloads ~75GB dataset)
cd clickbench && ./setup.sh [pg_version]

# Run benchmark
./run.sh [pg_version] [runs]
./run.sh pg18 3 --checkpoint --label=before-opt
./run.sh --query=42          # single query
```

### TPC-H

```bash
# First-time setup (generates and loads data)
cd tpch && ./setup.sh [pg_version] [scale_factor]

# Run benchmark
./run.sh [pg_version] [runs]
./run.sh pg18 3 --checkpoint --label=baseline
./run.sh --query=7
```

### Common flags

| Flag | Description |
|---|---|
| `--checkpoint` | Save results to `checkpoints/<hash>[-label]/` after run |
| `--checkpoint-only` | Archive current results without re-running |
| `--label=<text>` | Tag appended to checkpoint folder name |
| `--query=N` | Run only query N (1-based) |
| `--skip=N` | Skip query N |

Results are written to `checkpoints/current/` on every run (CSV + JSON + heatmap).
